# Language Learning System — Curriculum Structure

نظام بنيوي (Structure) لمشروع تعلّم لغات متعدد المسارات (language-independent)،
مبني على مبدأ **Mastery Learning** ومتوافق مع إطار **CEFR**.

هذا الريبو يحتوي على **الكود فقط**. محتوى الدروس الفعلي (JSON) **لا يُرفع** على
GitHub — يفضل محليًا على جهازك فقط. راجع قسم "فصل الكود عن البيانات" أسفل.

---

## البنية

```
langlearn/
├── backend/
│   ├── schema.sql          # تعريف قاعدة البيانات الكامل
│   ├── build_db.py         # يبني langlearn.db من ملفات data/*.json
│   ├── app.py              # Flask API (تصفح + إضافة محتوى)
│   ├── requirements.txt
│   └── data/                       # ⚠️ المحتوى الفعلي — محلي فقط، غير مرفوع
│       ├── locales.json
│       └── courses/
│           └── de-track/           # مثال: مسار تعلّم الألمانية
│               ├── course.json
│               └── A1/
│                   ├── level.json
│                   └── unit_01/
│                       ├── unit.json
│                       └── lesson_01.json
├── frontend/
│   ├── index.html
│   ├── css/style.css
│   └── js/{config,api,main}.js
└── .gitignore
```

## التشغيل (المرة الأولى)

### 1. تثبيت متطلبات الباك إند

```bash
cd backend
pip install -r requirements.txt
```

### 2. بناء قاعدة البيانات من ملفات JSON

```bash
python build_db.py
```

هذا بيقرا كل حاجة تحت `backend/data/courses/` ويبني `backend/langlearn.db`
من الصفر. **شغّل هذا الأمر في كل مرة تعدّل فيها ملفات JSON.**

### 3. تشغيل السيرفر (Backend)

```bash
python app.py
```

السيرفر هيشتغل على: `http://localhost:5050`
سيبه شغال في الـ terminal ده طول وقت استخدامك للواجهة.

### 4. فتح الواجهة (Frontend)

افتح ملف `frontend/index.html` مباشرة في المتصفح (دبل كليك عليه، أو
اسحبه على نافذة المتصفح). الواجهة هتتكلم تلقائيًا مع السيرفر على
`localhost:5050`.

> **ملاحظة:** لو حصلت مشاكل CORS غريبة من فتح الملف مباشرة (`file://`)،
> شغّل سيرفر ستاتيك بسيط بدلًا من الفتح المباشر:
> ```bash
> cd frontend
> python -m http.server 8080
> ```
> وبعدين افتح `http://localhost:8080` في المتصفح.

## الاستخدام اليومي (بعد أول إعداد)

كل مرة تيجي تشتغل على المشروع:

```bash
# Terminal 1 — شغّل الباك إند وسيبه شغال
cd backend
python app.py

# Terminal 2 (لو محتاج سيرفر استاتيك للفرونت إند)
cd frontend
python -m http.server 8080
```

بعد كده افتح `frontend/index.html` (أو `http://localhost:8080`) في المتصفح.

## إضافة محتوى جديد

### الطريقة 1: من الواجهة مباشرة (الأسهل)
افتح الواجهة → اختار Level → اضغط "+ إضافة وحدة" أو "+ إضافة درس" →
اكتب المحتوى بالـ 3 لغات → حفظ. التغييرات بتتحفظ فورًا في `langlearn.db`.

⚠️ **مهم:** التعديلات اللي تعملها من الواجهة بتتحفظ في `langlearn.db` مباشرة،
مش في ملفات JSON. لو عايز التعديلات دي تفضل كمصدر JSON (عشان لو احتجت
تبني القاعدة من الصفر تاني)، لازم تنسخها يدويًا لملف JSON مطابق تحت
`backend/data/courses/...`. هذا قيد معروف في النسخة الحالية — راجع
قسم "تحسينات مستقبلية" تحت.

### الطريقة 2: تحرير ملفات JSON مباشرة (الأدق للمحتوى الكبير)
1. أنشئ/عدّل ملف `unit.json` أو `lesson_XX.json` تحت المسار المناسب،
   باتباع الشكل الموضّح في `backend/data/README.md`.
2. شغّل `python build_db.py` لإعادة بناء القاعدة من الصفر.

## فصل الكود عن البيانات (لماذا؟)

محتوى الدروس هو الأصل الفكري الحقيقي للمشروع. الكود (schema, API,
واجهة) هو أداة عامة تقدر تشاركها أو ترفعها على GitHub بأمان، لكن
المحتوى التعليمي الفعلي يفضل خاص بيك. لهذا:

- `.gitignore` يستثني `backend/langlearn.db` وكل ملفات JSON تحت
  `backend/data/courses/` (باستثناء ملفات README التوضيحية).
- لو رفعت الريبو على GitHub، الكود هيبقى متاح للتتبع والنسخ
  الاحتياطية، لكن أي حد يعمل clone للريبو هيحتاج يجهّز بياناته
  الخاصة (أو تنسخها له يدويًا برّا Git).

## مرجع CEFR

ملف `backend/data/courses/de-track/A1/level.json` فيه قسم
`cefr_reference` بنصوص مأخوذة من:
- **Global Scale** (الجدول العام لمستويات CEFR)
- **Self-Assessment Grid** (شبكة التقييم الذاتي بالمهارات الخمس)

هذه النصوص **مرجع تحقق فقط** (validation reference) — تُستخدم للتأكد
إن outcomes الوحدات/الدروس اللي بنصممها متوافقة مع المعيار الرسمي،
وليست محتوى نولّده أو نعيد كتابته.
